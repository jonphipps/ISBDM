# has category of carrier

## Element Reference
<ElementReference frontMatter={frontMatter} />

#### Additional information

<div className="guid">
This element is based on the base carrier categories derived from the "Housing Format", "Intermediation Tool", and "Storage Medium Format" attributes of the <InLink href="https://www.rdatoolkit.org/archivedsite/docs/5chair10.pdf">RDA/ONIX Framework for Resource Categorization</InLink>.
</div>

<div className="guid">
Consider a manifestation that has sub-units that are intended to be manipulated in a game or puzzle as having the category of "object". Consider the sub-units that are pieces of the game or puzzle to be three-dimensional objects instead of the category of their pre- or post-manipulation states.
<details>
  <summary>Examples</summary>
  
    | Property | Value |
    |:---------|:------|
    | has category of carrier | "object" |
    
    *[Full example: <InLink href="/docs/fullex/fx068">Library (2022; Corner Piece Puzzles; object)</InLink>.]*
</details>
</div>

#### Stipulations

<div className="stip">
  Record at least one occurrence of the element.
</div>

<div className="stip">
  Use the element super-type to record a less specific attribute.
</div>

<div className="stip">
  Apply the stipulations for the element super-type unless indicated otherwise.
</div>

<div className="stip">
  Record a value that appears in the <InLink href="/docs/ves/1022">ISBDM Category of Carrier value vocabulary</InLink>, in the language or script of choice of the cataloguing agency.
  
  <details>
  <summary>Examples</summary>
  
  | Property | Value |
  |:---------|:------|
  | has category of carrier | "disque informatique" |
  
  *[Full example: <InLink href="/docs/fullex/fx040">Algerie a travers les ages (2000; Cerist; disque informatique; version française</InLink>. The language of cataloguing is French.]*
  </details>
</div>

<div className="stip">
  For a manifestation that is a super-unit, record the value that applies to the largest number of sub-units.
</div>

<div className="stip">
  For a manifestation that is a super-unit, and if no one value predominates, record separate occurrences of the element for each value that applies to the larger number of sub-units.
  
  <details>
  <summary>Examples</summary>
  
  | Property | Value |
  |:---------|:------|
  | has category of carrier | "volume" |
  | has category of carrier | "audio disc" |
  
  *[Full example: <InLink href="/docs/fullex/fx032">Mojo 357 (2023; H Bauer Publishing Ltd.; volume + audio disc)</InLink>.]*
  </details>
</div>
