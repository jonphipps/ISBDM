# has manifestation associated with manifestation

## Element Reference
<ElementReference frontMatter={frontMatter} />

#### Additional information

<div className="guid">
This element is symmetric: it is its own inverse.
A manifestation that has an associated manifestation is an associated manifestation of the other manifestation.
</div>

<div className="guid">
This element supports the user tasks to find and identify the manifestation and to explore the context of the manifestation and its relationship to other instances of the Manifestation entity.
</div>

#### Element values

<div className="guid">
The values of this element may be indexed for uncontrolled or controlled keywords, or used to construct browsing lists.
</div>

##### String encoding schemes

<div className="guid">
Use a string encoding scheme that is applied to a <InLink href="/docs/relationships/1257">has authorized access point of manifestation</InLink> element.
</div>

<div className="guid">
A string encoding scheme for an authorized access point of a manifestation should support several levels of identification and collation:
  1. a title that is assigned by an agent who is responsible for the creation of the manifestation or by a cataloguing agency
  2. a date, agent, and place of creation of the manifestation
  3. specific attributes of the manifestation
</div>

<div className="guid">
<InLink href="/docs/ves/ISBDMSESMan">Manifestation SES</InLink> is a string encoding scheme suggested by <span class="thisem">ISBD for Manifestation</span> for an authorized access point of a manifestation.
</div>
